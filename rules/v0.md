# Repo Triage Rubric — v0

Five factors, each scored 0–3 by a human reading the repo. Composite =
sum of the five factors (max 15). Each factor name below is the literal
key used in the scoring stub front-matter and the JSON schema.

## Factors

### thesis_still_credible — Thesis still credible

The why-it-matters statement in the repo's README would survive a
five-minute hostile read today.

- **0** — The thesis was never written down, or it has been falsified
  by something that happened in the last quarter.
- **1** — The thesis exists but reads as wishful or stale. A hostile
  reader would call it a hobby.
- **2** — The thesis survives a hostile read but the operator hesitates
  to ship it as-is.
- **3** — The thesis survives a hostile read and the operator would
  paste it into a funder note unchanged.

### pull_from_outside — Pull from outside

At least one external signal in the last 30 days (citation, issue,
inbound, real user). The signal must be from someone who is not the
operator.

- **0** — Nothing inbound. No issues, no citations, no users.
- **1** — A single weak signal (a star, a passing mention).
- **2** — At least one real inbound: a substantive issue, a citation,
  a real user touching the artifact.
- **3** — Multiple inbound signals from independent sources; the work
  is being pulled, not pushed.

### shippable_next_step — Shippable next step

A concrete next PR exists and is bounded to one to three weeks of work.

- **0** — No next PR. The operator could not name what would ship next
  if asked.
- **1** — The next step exists but is unbounded — "rewrite the
  architecture" or "explore X."
- **2** — A concrete next PR with a rough scope but no acceptance
  criteria written down.
- **3** — Concrete next PR with written acceptance criteria, bounded
  to one to three weeks.

### cost_of_freeze — Cost of freeze

Freezing this repo for a month does not foreclose a timely
opportunity. Higher score = freezing costs the operator less.

- **0** — Freezing forecloses something the operator already committed
  to (a deadline, an external promise, a dependent shipping plan).
- **1** — Freezing risks a soft commitment or a downstream waiter.
- **2** — Freezing is fine for one month; the world will wait.
- **3** — Freezing for a quarter is fine; nothing time-sensitive is in
  flight.

### dependency_in_portfolio — Dependency in the portfolio

Another repo in the portfolio depends on this one shipping a particular
thing in the next 90 days.

- **0** — No portfolio dependency, and the operator can name no place
  this repo's output is consumed.
- **1** — A loose dependency exists but no concrete artifact is being
  awaited.
- **2** — A named downstream artifact is awaited, but the deadline is
  open-ended.
- **3** — A named portfolio repo is blocked on a specific artifact from
  this repo with a deadline in the next 90 days.

## Composite

Composite = sum of the five factors. Range: 0–15.

The forcing function:

- Top 2 by composite → ATTEND (next month's shipping attention)
- Bottom 3 by composite → RETIRE (archived; revival requires a new
  written rationale in a subsequent memo)
- Everything else → FREEZE (still alive, no attention this month)

Ties are broken by the operator's written judgment, recorded inline in
the memo as a single sentence.

## Pinning

Every memo pins this rubric file by version in its front-matter
(``rubric_version: v0``). A rubric revision is a new file ``rules/vN.md``
with a bumped version. Past memos remain pinned to the rubric version
they were scored against.
